# 迁移到另一台服务器与复刻环境流程

目标：把当前教程目录同步到 `121.48.163.87:/data2/lyc/learn_pytorch`，并在目标服务器复刻 `eat-pytorch` conda 环境、Jupyter kernel 和 VS Code/Jupyter 使用方式。

## 1. 本机已导出的环境文件

当前目录下有几类文件：

- `environment-eat-pytorch.yml`：`conda env export -n eat-pytorch` 的完整导出，包含精确包版本，也包含本机 `prefix`。
- `environment-eat-pytorch-portable.yml`：删除了 `prefix` 的完整迁移版，推荐跨服务器使用。
- `environment-eat-pytorch-from-history.yml`：`conda env export --from-history` 的极简导出，只记录 conda 显式安装项。
- `environment-eat-pytorch-from-history-portable.yml`：删除 `prefix` 的极简迁移版。
- `requirements-eat-pytorch.txt`：`pip freeze` 结果，适合排查或手动补装。

推荐在目标服务器优先使用：

```bash
environment-eat-pytorch-portable.yml
```

如果目标服务器使用 `conda env create -f environment-eat-pytorch-portable.yml` 遇到 pip 对 `torch==2.11.0+cu128` 解析问题，则使用后文的“稳妥复刻方案”。

## 2. rsync 同步项目目录

在当前服务器执行：

```bash
rsync -avh --progress \
  /datadisk2/linyichen/eat_pytorch_in_20_days/ \
  121.48.163.87:/data2/lyc/learn_pytorch/
```

如果远端用户名不是当前用户名，需要写成：

```bash
rsync -avh --progress \
  /datadisk2/linyichen/eat_pytorch_in_20_days/ \
  用户名@121.48.163.87:/data2/lyc/learn_pytorch/
```

如果目标目录不存在，可先在远端执行：

```bash
ssh 121.48.163.87 'mkdir -p /data2/lyc/learn_pytorch'
```

再执行 rsync。

常用选项说明：

- `-a`：归档模式，保留目录结构、时间戳、权限等。
- `-v`：输出详细过程。
- `-h`：以人类可读单位显示文件大小。
- `--progress`：显示同步进度。
- 源路径最后的 `/` 表示同步目录内容，而不是把整个目录嵌套进去。

## 4. 推荐：稳妥复刻方案

这个方案比直接 `conda env create -f` 更稳，原因是 PyTorch CUDA wheel 使用了 PyTorch 官方 wheel index，普通 `pip freeze` 或 conda yml 在不同机器上有时不能完整表达这个 index。

在目标服务器执行：

```bash
cd /data2/lyc/learn_pytorch

conda create -n eat-pytorch python=3.10 -y
conda activate eat-pytorch

python -m pip install --upgrade pip setuptools wheel

python -m pip install \
  ipykernel numpy pandas matplotlib tqdm scikit-learn seaborn pillow \
  tensorboard jupyterlab notebook

python -m pip install \
  torch torchvision torchaudio \
  --index-url https://download.pytorch.org/whl/cu128

python -m pip install "torchkeras>=4.0.0" tabm

python -m ipykernel install \
  --user \
  --name eat-pytorch \
  --display-name "Python (eat-pytorch)"
```

如果目标服务器的 NVIDIA driver 太旧，不支持 CUDA 12.8 wheel，可换成 CUDA 12.6 或 11.8：

```bash
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu126
```

或：

```bash
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
```

## 7. VS Code 远程使用流程

在本地 VS Code 中：

1. 使用 Remote-SSH 连接 `121.48.163.87`。
2. 打开远端目录：

   ```text
   /data2/lyc/learn_pytorch
   ```

3. 确认远端 VS Code Server 已安装这些扩展：

   - Python
   - Jupyter

   注意：VS Code 扩展需要安装在远端 SSH 环境中，不只是本地装好。

4. 打开 `.ipynb` 文件。
5. 在 notebook 右上角选择：

   ```text
   Python (eat-pytorch)
   ```

6. 先运行检查单元：

   ```python
   import sys, torch
   print(sys.executable)
   print(torch.__version__)
   print(torch.version.cuda)
   print(torch.cuda.is_available())
   ```

如果 `sys.executable` 指向 `.../envs/eat-pytorch/bin/python`，并且 `torch.cuda.is_available()` 是 `True`，说明 notebook 已经使用正确环境。

## 8. notebook 中安装包的规则

在 `.ipynb` 中不要优先使用：

```python
!pip install package
```

更推荐：

```python
%pip install package
```

或者：

```python
import sys
!{sys.executable} -m pip install package
```

这样能确保包安装到当前 notebook kernel 对应的 Python 环境中。

## 9. 同步后的 Git 注意事项

如果只是学习和迁移，可以原样保留 `.git`。

如果你想让目标服务器上的目录变成自己的新 Git 仓库：

```bash
cd /data2/lyc/learn_pytorch
mv .git .git.backup-original
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <你的新仓库URL>
git push -u origin main
```

确认新仓库正常后，再删除备份：

```bash
rm -rf .git.backup-original
```
